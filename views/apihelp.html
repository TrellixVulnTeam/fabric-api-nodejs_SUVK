平台API帮助说明

一、   环境概述

1、  区块链底层节点

目前后台部署了三个智能合约，分属三个组织（A省公司，B省公司，C省公司），

每个组织下分别部署了2个peer节点（peer1和peer2）共6个peer节点。

2、  网络信息

内网IP：10.139.12.4

公网IP：223.105.0.146

端口：4000

注：以下API示例均采用外网IP

二、   登录注册API

2.1、注册

URL

xxx.xxx.xxx.xxx:xx/users/register

调用方式

POST

HTTP头部

content-type: application/x-www-form-urlencoded

HTTP内容

username={name}&password={password}&org={org}&phonenumber={phonenumber}

请求参数说明

username:要注册的用户名

password:设置的登录密码

org:组织名，在org1,org2,org3中选择,对应A、B、C三个省公司

phonenumber:注册手机号，用于找回密码

返回值

成功和失败均会返回对应弹窗

示例

要注册用户user1，密码123456，组织org1，手机号15906143346

url: 223.105.0.146:4000/user/register

http内容：username=user1&password=123456&org=org1&phonenumber=15906143346

2.2、   登录

URL

xxx.xxx.xxx.xxx:xx/users/login

调用方式

POST

HTTP头部

content-type: application/x-www-form-urlencoded

HTTP内容

username={name}&password={password}

请求参数说明

username:要登录的用户名

password:登录密码

返回值

成功返回：主页。同时包括{token:’token’,username:’username’,userorg:’userorg’}

失败返回：登录页，同时包括{loginerr:’errormessage’}

示例

要登录的用户user1，密码123456

url：223.105.0.146:4000/user/login

http内容：username=user1&password=123456

三、物联网日志API接口

3.1根据日志名获取日志信息

URL

xxx.xxx.xxx.xxx:xx/channels/:channelID/chaincodes/:chaincodeID

?peer={peerID}&fcn=readLog&args=%5B%22{name}%22%5D

调用方式

GET

HTTP头部

authorization: Bearer $ORG_TOKEN

content-type: application/json

HTTP内容

 

请求参数说明

channelID:查询的channel名，默认值：logchannel

chaincodeID:创建的智能合约名称，默认值：logcc

peerID:调用的节点名，默认值：peer1

fcn:指调用智能合约中的函数

readLog函数:根据某个日志名称，查找该日志对应的信息

name:日志名

authorization:身份验证，默认方式为Bearer，参数值为创建用户时返回的token值

返回值

{

“status code” : 200

“data” : {

           "name": " logtest",

           "logContent": "900150983cd24fb0d6963f7d28e16f72",

           "user": "tom"

         }

}

示例

要查询名为logtest的信息

url:223.105.0.146:4000/channels/logchannel/chaincodes/logcc?

peer=peer1&fcn=readLog&args=%5B%22logtest%22%5D

四、交易系统API接口

4.1获取用户商品信息

URL

xxx.xxx.xxx.xxx:xx/channels/:channelID/chaincodes/:chaincodeID?

peer={peerID}&fcn=queryItemsByItemOwner&args=%5B%22{itemname}%22%2c%22{user}%22%5D

调用方式

GET

HTTP头部

authorization: Bearer $ORG_TOKEN

content-type: application/json

HTTP内容

 

请求参数说明

channelID:创建的channel名，默认值：itemchannel

chaincodeID:创建的智能合约名称，默认值：itemcc

peerID:调用的节点名，默认值：peer1

fcn:指调用智能合约中的函数

queryItemsByItemOwner函数:根据商品名,用户名查找商品，参数商品名和用户名可为空，

若两者都为空，则返回当前用户下所拥有的全部商品，

若商品名为空，则返回查询用户下所有商品。

Itemname：商品名

user:用户名

authorization:身份验证，默认方式为Bearer，参数值为创建用户时返回的token值

返回值

{

“status code” : 200

“data” : {

         “Key”:” JimIPRIntellectual property right”,

           "Record": {

"docType":"item",

"name":"zhishi",

"owner":"Jim",

"price":10,

"property":"Intellectual property right"

                    }

         }

}

示例

查询名为itemtest的商品

url：223.105.0.146:4000/channels/itemchannel/chaincodes/itemcc?

peer=peer1&fcn=queryItemsByItemOwner&args=%5B%22itemtest%22%2c%22%22%5D
